# R3 RAG 성능 평가 실험 설계

## 1. 청킹 전략

### 평가 변수

- **청킹 단위:** 법령을 어떤 구조 단위(조 / 항 / 호 등)로 나눠서 벡터화할지 결정하는 기준
- **멀티단위 (멀티 그레뉼러리티):** 같은 법령을 서로 다른 크기의 단위(예: 항+호, 조+항+호)로 동시에 인덱싱하는 전략
- **prefix:** chunk 본문 앞에 법령명·조문번호 같은 구조 정보를 텍스트로 붙여 임베딩에 포함시키는 것
- **Top-k:** 검색 시 유사도가 높은 chunk를 몇 개까지 가져올지 정하는 값
- **hybrid:** 구조 단위로 자르되, 너무 길거나 짧으면 토큰 기준으로 추가 분할/병합하는 보정 방식
- **청킹 크기(min/max_tokens):** 하나의 chunk가 가질 최소·최대 토큰 길이를 제한하는 설정
- **overlap:** 인접 chunk 사이에 일부 문장을 겹치게 포함시켜 문맥 단절을 줄이는 방식

#### 주요 비교 분석

**C0 vs C6 (청킹 단위만 비교: 항 vs 조)**

| 지표      | C0 (paragraph) | C6 (article) | 차이     |
| --------- | -------------- | ------------ | -------- |
| MH-Recall | **0.4333**     | 0.0000       | -43.33%p |
| Recall    | **0.3389**     | 0.0111       | -32.78%p |
| MRR       | **0.4889**     | 0.0083       | -48.06%p |

> **결론**: 조(article) 단위 청킹은 완전히 실패. 청크가 너무 커서 세밀한 매칭 불가.

**C0 vs C1 (prefix + hybrid 효과)**

| 지표      | C0     | C1         | 차이        |
| --------- | ------ | ---------- | ----------- |
| MH-Recall | 0.4333 | **0.4833** | **+5.00%p** |
| Recall    | 0.3389 | **0.3500** | +1.11%p     |
| MRR       | 0.4889 | 0.4889     | 0%p         |

> **결론**: `prefix: article_only` + `hybrid: true` 추가로 MH-Recall +5%p 향상.

**C1 vs C2 (subparagraph 멀티 인덱싱 효과)**

| 지표      | C1         | C2         | 차이        |
| --------- | ---------- | ---------- | ----------- |
| MH-Recall | **0.4833** | 0.4667     | -1.66%p     |
| Recall    | 0.3500     | 0.3500     | 0%p         |
| MRR       | 0.4889     | **0.5094** | **+2.05%p** |
| 청크 수   | 3,196      | 5,869      | +84%        |

> **결론**: 호(subparagraph) 추가로 MRR 향상, but MH-Recall 소폭 하락. 청크 수 급증 대비 효과 미미.
