# PRD – 규제 샌드박스 신속확인 컨설턴트 지원 AI

---

## 1. 개요 (Overview)

**프로젝트명**

규제 샌드박스 신속확인 컨설턴트 지원 AI (Working Title)

**한 줄 요약 (Problem → Solution)**

비정형 서비스 설명과 파편화된 승인 사례로 인해 과도한 조사 부담을 겪는 규제 샌드박스 컨설턴트를 위해,

**유사 승인 사례 기반의 대상성 판단·트랙 추천·신청서 초안을 자동으로 생성하는 전문가용 AI 워크플레이스**

**프로젝트 설명**

본 프로젝트는 규제 샌드박스 신속확인 업무를 수행하는 전문 컨설턴트를 대상으로, 기업의 비정형 서비스 설명을 규제 판단이 가능한 구조로 정리하고, 유사 승인 사례를 기반으로 샌드박스 대상성 판단, 트랙 추천, 신청서 초안 작성을 지원하는 AI 서비스이다. 컨설턴트의 반복적인 자료 조사 및 문서 초안 작성 시간을 획기적으로 단축하고, 보다 전략적인 판단과 고객 커뮤니케이션에 집중할 수 있도록 돕는 것을 목표로 한다.

**작성일 / 버전**

- v1.0 (MVP 기준)

## 2. 문제 정의 (Problem Statement)

### 2-1. 왜 이걸 만들어야 하는가

규제 샌드박스는 신기술·신산업 서비스가 기존 법·제도 하에서 불가능하거나 불명확한 경우, 한시적 특례를 통해 실증과 제도 개선을 가능하게 하는 핵심 제도이다. 정부는 규제 혁신을 위해 절차를 간소화하고 심의 기간을 단축하고 있으며, 2026년 산업융합촉진법 개정 이후 신속확인 신청 수요는 더욱 증가할 것으로 예상된다.

그러나 이러한 제도 개선은 역설적으로 **신속확인 업무를 수행하는 컨설턴트의 업무 부담을 급격히 증가**시키고 있다.

### 2-2. 사용자가 겪는 핵심 문제

- 기업의 서비스 설명은 대부분 **비정형 텍스트**로 제공되어, 규제 판단 가능한 구조로 재정리하는 데 많은 시간이 소요됨
- 수백 건의 승인 사례가 PDF·웹·보도자료 등으로 **파편화**되어 있어, 유사 사례 탐색이 거의 수작업에 의존
- “이 서비스가 샌드박스를 검토할 가치가 있는가?”, “어떤 트랙이 적합한가?”에 대한 **초기 판단 근거를 빠르게 만들기 어려움**
- 신청서 초안은 반복적인 문체와 구조를 가지지만, 매번 새로 작성해야 함
- 짧아진 회신 기한(30일 → 15일) 대비, 분석·조사·문서 작업 시간은 줄어들지 않음

### 2-3. 기존 해결 방법의 한계

- 개인 노하우와 엑셀·문서 템플릿에 의존
- 승인 사례 검색은 키워드 검색 수준에 머무름
- ChatGPT 등 범용 LLM은 제도 이해·문체·근거 제시에 한계
- 팀/회사 차원의 히스토리 관리 및 재사용 불가

➡️ **문제의 핵심은 “기술적으로 가능하냐”가 아니라,
컨설턴트의 실제 업무 흐름에서 반복적으로 발생하는 구조적 비효율**이다.

## 3. 목표 & 성공 지표 (Goals & Metrics)

### 3-1. 목표 (Goals)

**정성적 목표**

- 컨설턴트의 조사·초안 작성 시간을 획기적으로 단축
- 샌드박스 대상성 판단의 초기 품질을 안정화
- 개인 경험에 의존하던 노하우를 데이터 기반으로 보조

**정량적 목표 (MVP 기준 가설)**

- 1건 Case 처리 초기 분석 시간 50% 이상 단축
- 초안 생성 기능 사용률 70% 이상
- 반복 사용 컨설턴트 비율(재방문) 40% 이상

### 3-2. KPI / Success Metrics

- DAU / MAU (컨설턴트 기준)
- Case 생성 → 초안 생성까지의 Task Completion Rate
- Case당 평균 처리 시간
- 초안 다운로드 비율
- NPS (컨설턴트 만족도)

## 4. 타겟 사용자 (Target Users)

### 4-1. 주요 사용자

**Primary User**

- 규제 샌드박스 신속확인 컨설턴트
- 소속: 컨설팅사, 법무법인, 정책 컨설팅 조직
- 역할: 기업 서비스 분석, 제도 적용 가능성 검토, 신청서 작성

**Secondary User (향후)**

- 기업 내 정책/전략 담당자
- 스타트업 대표 (전문가 검수 전 내부 검토용)

### 4-2. 사용자 페르소나

**직업**

- 정책·규제 컨설턴트 (경력 3~10년)

**사용 상황**

- 고객사 미팅 직후
- 비정형 서비스 설명과 제한된 시간
- 여러 Case를 동시에 처리

**Pain Point**

- “이 서비스가 샌드박스 대상인지 빠르게 판단해야 한다”
- “유사 사례를 찾는 데 너무 오래 걸린다”
- “초안은 매번 비슷한데 다시 쓰는 게 비효율적이다”

**니즈**

- 빠른 구조화
- 판단 근거가 있는 추천
- 수정 가능한 초안

## 5. 유저 시나리오 & 유저 플로우

### 5-1. 핵심 시나리오 (Happy Path)

1. 컨설턴트가 로그인 후 새 Case를 생성한다
2. 고객사 서비스 설명(신청서/미팅 노트)을 입력한다
3. AI가 서비스를 규제 판단 가능한 구조로 정리한다
4. AI가 샌드박스 대상성 1차 판단을 제시한다
5. 적합한 샌드박스 트랙을 추천한다
6. 신청서 초안을 자동 생성한다
7. 결과를 수정·저장하고 Case 히스토리로 관리한다

### 5-2. 주요 유저 플로우

로그인 → Case 생성 → 서비스 구조화 → 대상성 판단 → 트랙 추천 → 신청서 초안 생성 → Case 관리

## 6. 기능 요구사항 (Functional Requirements)

### 6-1. 핵심 기능 (MVP)

1. 서비스 설명 구조화 & 규제 쟁점 정리
2. 샌드박스 대상성 판단 보조 (1차 진단)
3. 샌드박스 트랙 추천 + 근거
4. 샌드박스 신청서 초안 자동 생성
5. 컨설팅 Case 관리

---

### 6-2. 상세 기능 정의

### [기능 1] 서비스 설명 구조화

- **설명**: 비정형 입력을 규제 판단 가능한 구조로 변환
- **입력**: 신청서 PDF, 미팅 노트, 자유 텍스트
- **출력**:
  - 서비스 한 줄 요약
  - 핵심 기능 분해
  - 잠재 규제 쟁점 후보
  - 관련 법령 후보
- **예외**:
  - 입력이 지나치게 짧을 경우 추가 정보 요청

---

### [기능 2] 샌드박스 대상성 판단 보조

- **설명**: 샌드박스 검토 가치에 대한 1차 진단 제공
- **입력**: 구조화된 서비스 정보
- **출력**:
  - ✔️ 필요성 높음 / ⚠️ 병행 검토 / ❌ 가능성 낮음
  - 판단 근거 요약
- **예외**:
  - 법적 결론이 아님을 명확히 표시

---

### [기능 3] 샌드박스 트랙 추천

- **설명**: 적합한 샌드박스 유형 추천
- **입력**: 대상성 판단 결과, 서비스 구조
- **출력**:
  - 추천 트랙(1~2개)
  - 제도 기준 기반 추천 사유

---

### [기능 4] 신청서 초안 자동 생성

- **설명**: 신청서 템플릿 기반 초안 생성
- **입력**: 구조화 정보, 선택 트랙, 부서별 템플릿
- **출력**:
  - 신청서 초안 (DOCX / PDF)
- **예외**:
  - MVP에서는 제한된 트랙/부서만 지원

---

### [기능 5] Case 관리

- **설명**: 컨설팅 히스토리 관리
- **저장**:
  - 구조화 결과
  - 판단 결과
  - 초안 버전
- **추가**:
  - 상태 관리
  - 버전 히스토리
